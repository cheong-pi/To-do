# 제품 규칙

## 최상위 규칙

1. 오늘 화면이 월간 화면보다 우선이다.
2. 오늘 화면의 기본 행동 버튼은 3개 이하로 유지한다.
3. 기본 버튼은 `[시작]`, `[미루기]`, `[취소]`이다.
4. 미룸과 연기는 실패가 아니라 기록과 재예약 신호다.
5. 완료는 자동 처리하지 않고 사용자에게 확인한다.
6. 선택지를 늘리기 전에 기본값으로 해결할 수 있는지 먼저 판단한다.
7. 기존 사용자 데이터 삭제는 금지한다.
8. 데이터 마이그레이션 없이 스키마를 변경하지 않는다.
9. 앱 이름은 `잊지 마`이고 PWA 짧은 이름도 `잊지 마`이다.
10. 메인 컬러는 `#FFE680`을 사용한다.
11. React 구현은 CSS Modules를 사용한다.
12. 주요 화면은 `할일`, `캘린더`, `단어`, `뽀모도로` 탭으로 관리한다.

## 화면별 규칙

### 오늘

- 앱의 첫 화면이다.
- 오늘 화면은 최대 15개까지 표시한다.
- 15개에는 자동으로 올라온 반복/기간/마감 일정과 사용자가 오늘에 추가한 일정이 모두 포함된다.
- 완료된 항목은 체크 표시하되 다시 되돌릴 수 있어야 한다.
- 해당 요일 반복 일정, 특정 기간으로 미리 설정한 일정, 마감 D-5부터의 일정은 자동으로 올라온다.
- 예정/언젠가 할 일은 사용자가 필요할 때 가져온다.
- 할일 리스트에는 할 일 내용과 상태만 간결하게 표시한다.
- 오늘 리스트에는 `계획`, `진행 중` 같은 상태 뱃지를 표시하지 않는다.
- 취소, 완료처럼 사용자가 알아야 하는 예외 상태만 시각적으로 구분한다.
- 리스트 안에 시작, 미루기, 취소, 진행률, 뽀모도로 조작을 직접 노출하지 않는다.
- 리스트 앞의 동그라미 체크를 누르면 해당 할 일을 완료 처리한다.
- 완료된 할일의 동그라미 체크를 다시 누르면 완료를 되돌리고 다시 노출한다.
- 각 할 일에는 `[수정]` 버튼을 제공한다.
- 수정 패널은 별도 사이드바가 아니라 해당 할일 바로 아래에 잠깐 펼쳐진다.
- 수정 패널은 제목, 날짜, 시간, 마감, 상태, 메모, 진행률을 직접 고치는 폼이다.
- 모바일 폭에서도 할일 리스트가 한 줄 흐름으로 내려가야 한다.
- 오늘 요약은 할일 리스트 위에 배치한다.
- `[할일 등록]`은 오늘 할일 제목 옆에 배치한다.
- `[수정]` 패널에는 시작 버튼을 두지 않는다.
- 할일 등록은 할일 탭 상단의 `[할일 등록]`에서 한다.
- 할일 등록 폼은 제목, 날짜, 시간, 마감, 날짜 없는 할일, 메모를 입력할 수 있어야 한다.
- 오늘 할일 화면은 좌우 화살표로 어제/오늘/내일 등 날짜를 이동할 수 있다.
- `시간 있음`, `마감 임박` 같은 내부 분류 문구는 리스트에 표시하지 않는다.
- 마감은 `D-5`, `D-1`, `D-Day`처럼 D-day만 표시한다.
- 가까운 D-day는 메인 노란색과 비슷한 명도/채도의 부드러운 빨간 배경으로 표시한다.
- 미루기 횟수는 리스트에 표시하지 않는다.
- 미룬 할일은 다음 알림 시간만 표시한다.
- 시간이 있는 할일은 `09:00`과 `09:00 알림`을 중복 표시하지 않고 `09:00 알림` 하나만 표시한다.
- `진행 중` 상태라고 해서 리스트 배경색을 파란색으로 바꾸지 않는다.
- 고정 할일로 등록한 항목은 할일 문장이 태그처럼 표시될 수 있다.
- 캘린더 색상은 할일 리스트 색상이 아니라 캘린더 표시 색상이다.
- 마감일은 D-day 계산용 날짜이다.
- 진행 기간은 해당 기간 동안 매일 오늘 할일에 노출할지 판단하는 날짜 범위이다.
- 등록/수정 폼에서 마감일(D-day)과 진행 기간 시작/종료는 별도 필드로 구분한다.
- 날짜 없는 할일을 선택하면 날짜, D-day, 진행 기간, 반복 설정은 비활성화한다.
- D-day를 선택하면 진행 기간과 반복 설정은 비활성화한다.
- 반복 설정을 선택하지 않으면 날짜와 시간만 기본 활성화하고 진행 기간/요일/매월 반복 세부 항목은 비활성화한다.
- 캘린더 색상은 `색상 지정` 체크박스를 켰을 때만 선택 가능하다.
- 반복 설정은 반복 없음, 매일, 요일 반복, 특정 기간, 매월을 지원한다.
- 요일 반복은 일/월/화/수/목/금/토 버튼으로 직접 선택한다.
- 반복 일정 수정 시 기본값은 `앞으로만 변경`이다.
- 반복 일정의 과거 완료/미룸/취소 기록은 규칙 변경으로 바꾸지 않는다.
- 15개 초과 시 다음 순서로 정렬해 상위 15개를 표시한다.

오늘 정렬 우선순위:

1. 시간이 정해진 오늘 일정
2. 사용자가 직접 오늘로 가져온 일정
3. 오늘 해당 반복 일정
4. 마감이 가까운 일정
5. 진행 중 일정
6. 날짜 없는 할 일

허용 버튼:

- 수정

금지:

- 첫 화면에 복잡한 필터를 과도하게 노출
- 리스트 항목에 여러 실행 버튼을 직접 노출
- 할일 리스트에 뽀모도로 설정 UI를 노출
- 시간 설정이 없는 할 일에 10분 미루기 버튼 노출
- 수정 패널에 시작 버튼 노출
- 할일 등록 위치를 숨기거나 찾기 어렵게 배치
- 미룸/연기 사용자에게 죄책감을 주는 문구

### 예정

- 날짜가 있는 예정 할 일, 마감기한이 있는 할 일, 날짜 없는 할 일을 보관한다.
- 오늘로 가져오기 기능을 제공한다.
- 오늘로 가져와 완료할 때 진행률을 입력할 수 있다.
- 진행률 입력은 `-` 버튼, 숫자 입력창, `+` 버튼으로 구성한다.
- `+`와 `-` 버튼은 10% 단위로 증가/감소한다.
- 숫자 입력창에는 0부터 100까지 직접 입력할 수 있다.
- 진행률이 100%이면 미래 원본은 완료 처리하고 오늘 반영으로 끝낸다.
- 진행률이 100% 미만이면 원본을 유지하고 남은 퍼센트를 표시한다.
- 오늘 화면을 대체하지 않는다.

### 월간

- 계획 입력보다 회고와 흐름 확인이 중심이다.
- 날짜별 완료, 미완료, 미룸 기록을 표시한다.
- 월간 화면에서 과도한 편집 기능을 우선 구현하지 않는다.

### 단어

- 사용자가 단어를 직접 입력하지 않는다.
- 기본 단어 세트는 초등-중등 영어 수준 3000개를 목표로 한다.
- 하루 학습 흐름은 20개 학습 후 질문/복습 단계로 넘어간다.
- 틀린 단어는 같은 학습 흐름 안에서 반복 출제한다.
- 복습 버튼은 `[알겠음]`, `[헷갈림]`, `[모름]`이다.

### 기록

- 이번 달 완료율, 미룸/연기 횟수, 취소 수, 못한 일, 외운 단어 수를 표시한다.
- 분석은 나중에 확장하되, 이벤트 기록은 MVP부터 남긴다.

## 상태 규칙

| 상태 | 의미 |
|---|---|
| 계획 | 아직 시작하지 않은 일정 |
| 진행 중 | 사용자가 시작을 누른 일정 |
| 완료 | 오늘 처리했거나 진행률 기준으로 완료된 일정 |
| 연기 | 해야 하는 일정이지만 오늘 처리하지 못해 남은 일정 |
| 취소 | 사용자가 하지 않기로 결정해 일정에서 제외한 항목 |

연기와 취소의 차이:

- 연기는 해야 하는 일정이라 나중에 다시 해야 한다.
- 취소는 일정에서 제외하고 하지 않을 예정이다.

## 미루기/연기 규칙

| 미루기 횟수 | 기본 동작 |
|---:|---|
| 1 | 10분 뒤 다시 알림 |
| 2 | 10분 뒤 다시 알림 |
| 3 | 10분 뒤 다시 알림 |
| 4 이상 | 연기 상태로 전환 |

- 하루 최대 3회까지 미룰 수 있다.
- 3회를 초과하면 연기 상태가 되지만, 당일 다시 진행할 수 있도록 오늘의 잔여 일정에 남긴다.
- 같은 할 일이 오늘과 내일 모두 있는 경우, 오늘 못한 일정은 취소 상태로 바꾸고 다음날 일정은 일반 계획 상태로 유지한다.
- 하루만 있는 할 일을 오늘 못한 경우, 날짜가 바뀌어도 오늘/내일 영역에 연기 상태로 남긴다.

- 지금 할 수 있는 만큼만 해요.
- 시작이 어려운 일 같아요.

## 뽀모도로 타이머 규칙

- 뽀모도로는 별도 탭에서 관리한다.
- 할일 리스트 안에 뽀모도로 입력 UI를 넣지 않는다.
- 집중 시간, 휴식 시간, 반복 횟수는 그때그때 선택할 수 있어야 한다.
- 고정값은 없다.
- 마지막으로 사용한 집중 시간, 휴식 시간, 반복 횟수는 설정에 저장한다.
- 예시 기본값은 25분 집중, 5분 휴식, 4회 반복이다.
- 집중 시간이 끝나면 `[휴식하기]`, `[멈춤]`을 보여준다.
- 휴식이 끝나면 `[집중 시작]`, `[멈춤]`을 보여준다.
- 설정한 반복 횟수가 끝나면 `[1세트 더 반복하기]`, `[멈춤]`을 보여준다.
- 멈춤을 누르면 타이머는 리셋된다.
- 타이머 종료 후 자동 완료하지 않는다.
- 뽀모도로 실행 세션은 서버에 저장하지 않는다.

## 알림 규칙

- 앱이 열려 있을 때는 정확한 타이머 알림을 제공한다.
- 설치된 PWA에서는 가능한 범위에서 브라우저 알림을 제공한다.
- 할 일 알림 문구는 할 일 내용 중심으로 표시한다.
- 할 일 시작 시간 기준으로 알림을 예약한다.
- 미루기를 선택하면 10분 뒤 재알림을 추가 저장한다.
- 알림 액션은 가능한 경우 시작, 미루기, 취소를 제공한다.
- 시작 액션은 진행 중 상태로 전환한다.
- 미루기 액션은 10분 뒤 재알림을 설정한다.
- 취소 액션은 해당 일정을 취소 상태로 전환한다.
- 웹만으로 완전한 네이티브 알람을 보장하지 않는다.
- Android/iOS 래퍼는 후속 검토 항목이다.

## 초기 설정 규칙

첫 실행 시 한 번만 묻는다.

| 설정 | 기본값 |
|---|---|
| 하루 할 일 목표 | 15개 |
| 하루 단어 수 | 20개 |
| 알림 시작 시간 | 09:00 |
| 기본 다시 알림 | 10분 뒤 |
| 마지막 집중 시간 | 25분 |
| 마지막 휴식 시간 | 5분 |
| 마지막 반복 횟수 | 4회 |

이후 설정 화면에서 변경 가능해야 한다.
